2. Les Composants de l'ArchitectureA. Calcul et Serveur d'Application (Compute)Cloud Run : Héberge vos conteneurs de manière serverless. Il s'adapte automatiquement au trafic (scaling à zéro si nécessaire).Configuration : Déployez votre application dans un VPC Connector (Serverless VPC Access) pour lui permettre de communiquer avec la base de données via une IP privée.B. Stockage des Fichiers (Storage)Cloud Storage (GCS) : Un bucket pour vos fichiers statiques (images, PDF, uploads).Sécurité : Désactivez l'accès public direct. Utilisez des Signed URLs (URLs signées) pour permettre aux utilisateurs de télécharger/uploader des fichiers de manière temporaire et sécurisée.C. Base de Données (Database)Cloud SQL (PostgreSQL ou MySQL) : Instance managée pour vos données structurées.Réseau : Configurez-la avec une IP privée uniquement. Elle ne sera accessible que depuis le VPC, protégeant ainsi vos données d'Internet.D. Réseau et Sécurité (Networking & Security)C'est ici que l'architecture devient "Enterprise Ready" :Cloud DNS : Gère votre nom de domaine (ex: api.votreentreprise.com).Cloud Load Balancing (HTTPS) : Sert de point d'entrée unique. Il gère les certificats SSL (gérés par Google) et répartit le trafic.Cloud Armor : Pare-feu applicatif (WAF) adossé au Load Balancer pour bloquer les attaques DDoS, les injections SQL et le Cross-Site Scripting (XSS).VPC & Subnets : Un réseau virtuel isolé pour vos ressources.3. Flux de Communication (Data Flow)L'utilisateur interroge le nom de domaine via Cloud DNS.La requête arrive sur le Cloud Load Balancer protégé par Cloud Armor.Le Load Balancer transfère la requête au service Cloud Run.Cloud Run récupère les fichiers sur Cloud Storage via l'API interne de Google.Cloud Run accède à la base de données Cloud SQL via le VPC Connector en utilisant une adresse IP interne (sans jamais sortir sur l'Internet public).4.
